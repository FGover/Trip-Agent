# Reporting

## Compact Report Shape

Use this shape when reporting evaluation results:

- Input source: JSON file, JSONL file, or Redis trip id.
- Rule evaluation: schema pass, hard pass, soft pass, budget consistency, and location validity.
- Judge evaluation: preference satisfaction, practicality, grounding faithfulness, budget reasonableness, coherence, and overall quality.
- Key issues: show only the top blocking errors or warnings.
- Next action: fix planner prompt, retrieval, candidate grounding, budget calculation, or frontend rendering contract.

## Resume-Safe Wording

构建项目级 Skill 工作流，规范 Planner 数据生成、规则评估与 LLM-as-Judge 评估流程，从结构合法性、工具结果忠实度、预算合理性和偏好满足度等维度支持生成效果迭代。

## What Not To Claim

Do not claim training improvements or online A/B gains unless the same evaluation set and comparable reports exist.
